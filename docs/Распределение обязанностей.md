### **Исполнитель 1: Backend (Логика)**
*Отвечает за всю не-визуальную часть: обработку данных, алгоритмы, вычисления.*

1.  **Модуль сессии тренировки:**
    *   Реализовать логику обработки вводимого текста: сравнение с оригиналом, обработка нажатий (включая Backspace), отслеживание ошибок.
    *   Рассчитывать базовые метрики в реальном времени и по завершении: скорость (WPM/CPM), точность.

2.  **Модуль анализа N-грамм:**
    *   Собирать данные о скорости и ошибках для каждой n-граммы (1-3 символа) во время сессии.
    *   Разработать алгоритм, который на основе собранной статистики определяет "слабые" n-граммы.
    *   Реализовать функцию подбора слов из словаря, которые содержат эти "слабые" n-граммы.

3.  **Модуль данных пользователя:**
    *   Создать систему для сохранения и загрузки прогресса пользователя (статистика n-грамм, история тренировок) в локальный JSON-файл.

---

### **Исполнитель 2: Frontend (Интерфейс на Qt)**
*Отвечает за все визуальные компоненты и взаимодействие с пользователем.*

1.  **Основной интерфейс тренировки:**
    *   Создать главное окно приложения.
    *   Разработать виджет для отображения текста с динамической подсветкой (правильно/неправильно набранные символы, текущая позиция).
    *   Обеспечить ввод текста и передачу нажатий в Backend.

2.  **Интеграция режимов:**
    *   Реализовать UI для выбора режима: "свой текст" или "умная тренировка".
    *   Отображать метрики (скорость, точность), получаемые от Backend, в реальном времени и по итогам сессии.

3.  **Экран статистики:**
    *   Создать отдельное окно или вкладку для отображения данных из профиля пользователя: история результатов, проблемные n-граммы.

---

### **Общая работа**

1.  **Проектирование API (в начале):**
    *   Совместно определить, какие функции и структуры данных Backend будет предоставлять для Frontend. Это — контракт между двумя частями проекта.
2.  **Интеграция и тестирование (постоянно):**
    *   Регулярно соединять Backend и Frontend, проверять корректность их совместной работы. Настроить сборку проекта через CMake.
